# UI-designregler – PayBySharePay

## Designretning

UI skal være inspireret af referencebillederne:
- moderne mobilapp-look
- store afrundede kort
- tydelig bundnavigation
- god spacing
- høj læsbarhed
- touch-venligt layout

## Responsive krav

- mobil-først
- optimeret til iPhone 14 bredde
- ingen overfyldte layouts
- knapper og inputfelter skal være nemme at trykke på

## Vigtige komponenter

- bundnavigation
- søgefelt
- kort/grid på forsiden
- lister over deltagere
- badges for deltager-type
- statuschips for afventer/betalt/accepteret
- primære CTA-knapper

## Find deltagere

Siden skal visuelt kunne vise forskel på:
- person
- merchant

Forslag:
- badge
- lille ikon
- sekundær tekst med virksomhedsnavn eller type

## Ordreoverblik

Skal vise:
- ordretype/kategori
- besked
- deltagerliste
- status
- betalingshandling

## Designregler for kode

- brug genanvendelige Angular components hvor det giver mening
- hold styling konsekvent
- undgå desktop-tunge layouts i første version
