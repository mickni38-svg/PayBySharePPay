# Use cases og acceptance criteria

## Use case 1 – Søg deltagere
Som bruger vil jeg kunne søge efter personer og merchants, så jeg kan vælge relevante deltagere til en ordre.

### Acceptance criteria
- resultater viser både personer og merchants
- deltager-type vises tydeligt
- søgning kan filtrere på navn eller relevant info

## Use case 2 – Opret merchant
Som administrator eller relevant bruger vil jeg kunne oprette en merchant med firma- og betalingsoplysninger.

### Acceptance criteria
- merchant kræver firmanavn
- merchant kan gemmes i databasen
- merchant kan findes i deltagersøgning bagefter

## Use case 3 – Opret ordre
Som bruger vil jeg kunne oprette en ordre og vælge deltagere.

### Acceptance criteria
- ordre kan gemmes
- deltagere kan knyttes til ordren
- ordre kan ses i overblik

## Use case 4 – Se ordreoverblik
Som bruger vil jeg kunne se status på en ordre.

### Acceptance criteria
- ordretype vises
- deltagere vises
- status vises pr. deltager
- betalingshandling vises

## Use case 5 – Registrer betaling
Som bruger vil jeg kunne registrere betaling på en ordre.

### Acceptance criteria
- gyldigt beløb kræves
- betalingsstatus opdateres
- betaling kan ses ved genindlæsning
